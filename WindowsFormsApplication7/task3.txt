Задание 3: создать программу с одним главным потоком и тремя вторичными потоками. 
Главный поток считывает пары (не пару, а пары) дискретных значений (x,y) из файла. 
Количество этих пар должно определяться автоматически по количеству строк в файле, то есть файл выглядит таким образом:

x1 y1
x2 y2
x3 y3
...

где вместо x1 y1, x2 y2 и так далее стоят конкретные числа типа double. 
Эти пары должны главным потоком считываться в глобальный массив. 
В интерфейсе пользователя необходимо запрашивать три степени сплайна (m n k),
где под степенью сплайна понимается его порядок (первого порядка = 1, второго порядка = 2, кубический = 3  и так далее).
Эти три числа должны считываться главным потоком в соответствующие глобальные переменные. 
Первый вторичный поток должен выполнять интерполяцию сплайном порядка m, второй - интерполяцию сплайном порядка n, 
третий - интерполяцию сплайном порядка k. По результатам работы трех вторичных потоков, записываемых в глобальные переменные, 
должны строиться графики трех сплайнов, а также маркерами должны рисоваться исходные значения по парам (x,y) из файла.

С помощью критических секций и функций ожидания должна быть организована последовательная работа (один за одним) 
трех вторичных потоков в порядке m-n-k с перебором всех возможных сочетаний приоритетов этих потоков. 
Необходимо оценить работу каждого из вторичных потоков с помощью функций GetThreadTimes и всего процесса в целом с помощью функции 
GetProcessTimes.
